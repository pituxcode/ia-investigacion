Estructurar documento con Markdown

Duración: 25 minutos

Objetivo

Aprender a usar Markdown para estructurar documentos académicos de forma eficiente, aprovechando la compatibilidad con herramientas de IA.

¿Por qué Markdown?

Markdown es un lenguaje de marcado simple que:

  • Es texto plano (funciona en cualquier editor)
  • Se convierte fácilmente a Word, PDF, HTML
  • Es ideal para trabajar con IA (formato que los modelos entienden bien)
  • Separa contenido de formato (te enfocas en escribir)
NotaMarkdown vs. Word
Aspecto Word Markdown
Formato Visual (WYSIWYG) Texto con marcas
Colaboración con IA Copiar/pegar fragmentos Flujo directo
Control de versiones Difícil Fácil (es texto)
Exportación Limitada Múltiples formatos

Texto plano: la base de todo

Hay una razón de fondo por la que Markdown importa para la investigación con IA: los modelos de lenguaje operan sobre texto plano. Cuando trabajas en Word, la IA no puede leer tu documento directamente — necesitas copiar fragmentos, pegarlos en un chat y volver a copiar el resultado. Cuando trabajas en texto plano, la IA puede leer, editar y generar contenido dentro de tu mismo flujo de trabajo.

Esto no es solo una conveniencia técnica. Es un cambio de paradigma: el texto plano se convierte en la interfaz común entre tu escritura, la IA, el cómputo científico y la publicación final.

Quarto: de texto plano a publicación

Quarto es una herramienta open source que convierte archivos de texto plano (.qmd) en documentos publicables. Un mismo archivo puede generar un PDF para la revista, un documento Word para tus coautores y un sitio web para difusión — sin cambiar nada del contenido.

El flujo es: escribes en Markdown, ejecutas código si lo necesitas (R, Python, Julia), y Quarto produce el formato final. Esta guía que estás leyendo fue creada exactamente así.

El editor como centro de operaciones

Herramientas como VS Code o Cursor permiten trabajar con texto plano y tener asistencia de IA integrada en el mismo entorno. Esto significa que puedes:

  • Redactar tu manuscrito en Markdown
  • Pedir a la IA que revise, traduzca o mejore secciones — sin salir del editor
  • Ejecutar análisis estadísticos en R o Python dentro del mismo documento
  • Generar PDF, Word y HTML con un solo comando
  • Controlar versiones con Git

Todo desde una herramienta open source, con texto plano como base. No dependes de licencias, formatos propietarios ni plataformas que pueden cambiar sus condiciones.

LaTeX, ecuaciones y referencias

Quarto tiene integración completa con LaTeX. Puedes escribir ecuaciones directamente en tu documento. Por ejemplo, esto:

$$
y = \beta_0 + \beta_1 x_1 + \beta_2 x_2 + \epsilon
$$ {#eq-regresion}

Se renderiza automáticamente como:

\[ y = \beta_0 + \beta_1 x_1 + \beta_2 x_2 + \epsilon \tag{3.1}\]

Las ecuaciones se numeran y puedes referenciarlas en el texto con @eq-regresion: “como se observa en la Ecuación 3.1, los coeficientes…”. Esto funciona igual en HTML, PDF y Word.

Para las referencias bibliográficas, Quarto usa archivos .bib (el mismo formato que LaTeX y Zotero). Defines tus fuentes en un archivo .bib y las citas en el texto con [@autor2025]. Quarto genera la bibliografía automáticamente en el formato que necesites (APA, Vancouver, Chicago, etc.).

NotaEjemplo real

Esta misma guía gestiona sus referencias así: el archivo references.bib contiene las fuentes y en el texto se citan con [@kalai2025hallucinate]. Quarto genera la sección de Referencias en formato APA automáticamente. Si exportas a Word o PDF, las citas se adaptan al formato correspondiente.

Cómputo científico integrado

Quarto no es solo un formato de documento: también puede ejecutar código. Puedes incluir bloques de Python, R o Julia directamente en tu manuscrito, y Quarto ejecuta el código y presenta los resultados — tablas, gráficos, estadísticas — como parte del documento final.

Por ejemplo, un bloque como este en tu archivo .qmd:

```{python}
import numpy as np

datos = [23.1, 25.4, 22.8, 24.6, 23.9, 26.1]
print(f"Media: {np.mean(datos):.2f}")
print(f"Desviación estándar: {np.std(datos):.2f}")
```

Produce directamente en el documento:

Media: 24.32
Desviación estándar: 1.18

Esto significa que tus resultados nunca se desactualizan: si cambian los datos, el documento se regenera con los valores correctos. No más copiar números de una planilla a un Word.

NotaPara profundizar

La guía oficial de Markdown en Quarto cubre toda la sintaxis disponible para documentos académicos: ecuaciones, referencias cruzadas, citaciones, figuras con subtítulos, tablas, código ejecutable y más.

Consulta: quarto.org/docs/authoring/markdown-basics.html

Paso 2: Mejorar la redacción

Una vez tengas el borrador, pide mejoras específicas:

Revisa este texto académico y mejora:
1. Claridad: simplifica oraciones complejas
2. Precisión: elimina ambigüedades
3. Fluidez: mejora las transiciones entre párrafos
4. Concisión: elimina redundancias

Mantén el tono académico formal. No cambies el significado.

[PEGA TU TEXTO]

Paso 3: Verificar consistencia de estilo

Revisa este texto y asegura consistencia en:
- Uso de tiempos verbales (pasado para métodos y resultados)
- Persona gramatical (tercera persona o impersonal)
- Terminología (usa siempre los mismos términos para conceptos clave)
- Formato de números y porcentajes

[PEGA TU TEXTO]

Errores comunes a evitar

AdvertenciaAtención
Error Por qué es problema Solución

Sobre-editar El texto pierde tu voz Pide cambios específicos, no generales

Traducción al inglés académico

Duración: 30 minutos

Objetivo

Traducir tu manuscrito al inglés académico manteniendo precisión terminológica y estilo apropiado para publicación internacional.

¿Por qué no usar Google Translate directamente?

Google Translate IA con prompt especializado
Traducción literal Adaptación al registro académico
No considera contexto disciplinar Usa terminología de tu área
Puede perder matices Mantiene el argumento
Sin control de estilo Puedes especificar convenciones

El prompt de traducción académica

Actúa como traductor especializado en textos académicos de
[TU DISCIPLINA].

Traduce el siguiente texto del español al inglés académico.

Instrucciones:
1. Usa terminología estándar de [TU DISCIPLINA] en inglés
2. Mantén el registro formal académico
3. Adapta expresiones idiomáticas al equivalente en inglés académico
4. Conserva la estructura argumentativa del original
5. Si hay términos que podrían traducirse de varias formas,
   usa el más común en revistas de alto impacto

No traduzcas literalmente. Produce un texto que suene natural
para un lector académico anglófono.

Refinamiento de la traducción

Después de la traducción inicial:

Revisa esta traducción al inglés académico y mejora:

1. Naturalidad: ¿suena como lo escribiría un nativo?

3. Convenciones: ¿sigue las normas del inglés académico?
   - Hedging apropiado (may, might, suggest, appear to)
   - Voz pasiva donde corresponde
   - Conectores formales

Prompt de revisión final

Usa este prompt para una revisión automatizada:

Actúa como editor de una revista académica de [TU ÁREA].

Revisa este manuscrito y genera un informe con:

1. ESTRUCTURA
   - ¿Están todas las secciones requeridas?
   - ¿La organización es lógica?

2. CLARIDAD
   - ¿Hay oraciones confusas o ambiguas?
   - ¿Los argumentos son fáciles de seguir?

3. CONSISTENCIA
   - ¿Los tiempos verbales son consistentes?
   - ¿La terminología es consistente?
   - ¿Las abreviaturas están definidas en primer uso?

4. POSIBLES PROBLEMAS
   - ¿Hay afirmaciones sin respaldo?
   - ¿Hay saltos lógicos?
   - ¿Falta información importante?

5. RECOMENDACIONES PRIORITARIAS
   - Lista las 3-5 mejoras más importantes

[PEGA TU MANUSCRITO]

  • Terminal

  2. Revisa el informe generado
  3. Haz las correcciones manualmente
  4. Pide una segunda revisión de las secciones modificadas

Declaración de uso de IA

Si usaste IA en la preparación del manuscrito, incluye una declaración como esta:

## Declaration of Generative AI Use

During the preparation of this manuscript, the authors used
[HERRAMIENTA] ([VERSIÓN]) for [PROPÓSITO: improving language
clarity / translating from Spanish to English / organizing
the literature review structure].

After using this tool, the authors reviewed and edited the
content as needed and take full responsibility for the
content of the publication.
ImportanteUbicación de la declaración
Editorial Ubicación
Elsevier Antes de las referencias
Springer Nature En la sección de Métodos
Taylor & Francis En Acknowledgments o según instrucciones

Siempre revisa las instrucciones específicas de la revista.
